# ToDo 웹 애플리케이션 요구사항 정의서

본 문서는 개발할 ToDo 웹 애플리케이션의 핵심 기능과 디자인 방향성, 그리고 데이터 구조를 한눈에 이해할 수 있도록 쉽게 정리한 공식 요구사항 정의서(PRD)입니다.

---

## 1. 프로젝트 개요

사용자가 일상의 업무와 학업 일정을 효율적으로 계획하고 관리할 수 있도록 돕는 웹 서비스입니다. 단순히 할 일을 받아 적는 메모장에서 벗어나, 우선순위와 카테고리별로 일정을 분류하여 오늘 꼭 처리해야 할 일에 집중할 수 있도록 돕는 시스템을 구축합니다.

* **비유로 이해하기**:
  이 프로그램은 사용자의 하루를 관리해 주는 '개인 비서' 역할을 합니다. 해야 할 일들을 중요도에 따라 분류해 주고, 기한이 언제까지인지 계속해서 리마인드해 줍니다.

---

## 2. 기술 스택 정의

개발 생산성을 극대화하고 추후 실제 서비스 배포까지 무리 없이 이어질 수 있도록 검증된 기술들을 사용합니다.

* **프로그램 뼈대 (Next.js)**:
  사용자가 웹사이트를 켰을 때 화면이 빠르게 뜨도록 돕는 현대적인 웹 개발 프레임워크입니다.
* **화면 디자인 (Tailwind CSS)**:
  디자인 작업을 빠르고 일관되게 진행할 수 있게 도와주는 스타일 도구입니다. 화면이 깔끔하고 정돈된 느낌을 주도록 돕습니다.
* **데이터 저장소 (SQLite & Neon DB)**:
  * 개발 단계(로컬): 사용자의 컴퓨터에 파일 형태로 가볍게 저장되는 로컬 데이터베이스(SQLite)를 사용합니다.
  * 운영 단계(추후): 개발이 끝나면 클라우드 환경의 데이터베이스(Neon DB)로 쉽게 전환하여, 여러 기기에서 접속해도 동일한 데이터를 볼 수 있도록 확장합니다.

---

## 3. 사용자 이용 흐름 (User Flow)

사용자는 아래와 같은 흐름으로 서비스를 이용합니다.

1. **회원가입 및 로그인**: 계정을 생성하고 로그인하여 개인 작업 공간을 활성화합니다.
2. **할 일 등록**: 본인만의 할 일이나 학업 일정을 등록하고 마감 기한과 우선순위를 지정합니다.
3. **필터링 및 확인**: 개인 할 일 목록에서 오늘 할 일, 미완료된 일 등을 골라내어 집중해서 처리합니다.
4. **상태 변경 및 피드백**: 할 일을 끝내면 체크박스를 눌러 완료 표시를 하고, 진행률 게이지를 통해 하루 성과를 확인합니다.

---

## 4. 기능적 요구사항 (사용자 기능)

### 4.1. 사용자 인증 (Authentication & Authorization)
* **회원가입**: 이메일 주소와 비밀번호를 입력하거나 소셜 계정(Google, GitHub 등)을 연동하여 가입할 수 있습니다.
* **로그인/로그아웃**: 가입한 계정으로 로그인하여 세션을 유지하고, 로그아웃하여 세션을 파기합니다.
* **접근 제어**: 로그인하지 않은 사용자는 서비스 메인 페이지에 접근할 수 없으며, 로그인 페이지로 리다이렉트됩니다.

### 4.2. 할 일 관리 (ToDo CRUD - 사용자 개인화)
* **등록**: 로그인한 사용자는 자신만의 할 일 제목(필수 입력), 상세 설명(선택), 마감 기한, 카테고리를 설정하여 등록합니다.
* **조회 및 검색**: 사용자는 **본인이 작성한 할 일 목록만** 조회하고 검색할 수 있으며 타인의 목록에는 접근할 수 없습니다.
* **수정 및 삭제**: 본인이 작성한 할 일에 한해서만 완료 여부 토글, 내용 수정, 개별 삭제 및 일괄 삭제가 가능합니다.

### 4.3. 분류 및 우선순위
* **카테고리**: '공부', '취미', '약속' 등 사용자가 직접 이름을 붙여 분류할 수 있는 태그 기능을 제공합니다.
* **우선순위**: 할 일의 중요도에 따라 '높음(빨강)', '보통(노랑)', '낮음(회색)' 3단계로 지정할 수 있어 시각적인 구분을 돕습니다.

### 4.4. 데이터 보존 및 보안
* 모든 할 일 데이터는 데이터베이스 내에서 사용자 식별자(UserID)와 매핑되어 영구적으로 보존됩니다.
* 비밀번호 등 민감한 개인 정보는 단방향 암호화(해싱)되어 안전하게 보존됩니다.

## 5. 서비스 품질 및 디자인 요구사항 (비기능 사양)

* **사용자 경험 및 애니메이션**:
  * 마우스 커서를 올렸을 때 버튼의 색상이 부드럽게 변하는 효과를 줍니다.
  * 할 일을 완료 처리하면 텍스트에 부드럽게 취소선이 그어지는 시각적 연출을 더합니다.
* **모바일 및 PC 최적화 (반응형 웹)**:
  * 스마트폰 화면처럼 좁은 곳에서는 화면 요소들이 세로로 길게 정렬됩니다.
  * 모니터 화면처럼 넓은 곳에서는 요소들이 가로로 넓게 배치되어 공간을 효율적으로 사용합니다.
* **성능 안정성**:
  * 사용자가 글자를 타이핑할 때 화면이 버벅거리지 않도록 불필요한 새로고침 연출을 차단하여 최적의 브라우저 사용 환경을 제공합니다.

## 6. 데이터 구조 설계

관계형 데이터베이스(PostgreSQL/Neon DB) 내에서 관리할 데이터 구조는 아래와 같습니다.

### 6.1. User (사용자 테이블)
1. **아이디 (ID)**: 고유 식별자 (CUID 또는 UUID)
2. **이메일 (Email)**: 로그인 ID로 사용될 고유 이메일 주소
3. **비밀번호 (Password)**: 단방향 해시 암호화된 비밀번호 (소셜 로그인 가입 시 Null 가능)
4. **생성일 (CreatedAt)**: 계정 생성 날짜와 시간

### 6.2. Todo (할 일 테이블 - User와 1:N 관계)
1. **고유 아이디 (ID)**: 각 할 일을 구분하는 고유 식별자 (CUID)
2. **사용자 아이디 (UserID)**: 작성한 사용자의 고유 식별자 (외래 키)
3. **할 일 제목 (Title)**: 한 글자 이상 작성해야 하는 텍스트
4. **상세 설명 (Description)**: 할 일에 대해 덧붙이는 부연 설명
5. **완료 상태 (IsCompleted)**: 완료되었는지 아직 진행 중인지 나타내는 상태 (참/거짓)
6. **우선순위 (Priority)**: 높음, 보통, 낮음 중 하나
7. **카테고리 (Category)**: 할 일이 속한 그룹의 이름 (예: 일반, 공부 등)
8. **마감 기한 (DueDate)**: 일을 끝내야 하는 날짜 정보
9. **등록일 (CreatedAt)**: 최초로 할 일을 작성한 날짜와 시간
10. **수정일 (UpdatedAt)**: 마지막으로 할 일을 편집한 날짜와 시간

---

## 7. 향후 추가 및 발전 계획

* **모바일 푸시 알림**: 마감 기한이 얼마 남지 않은 일정이 있을 경우, 브라우저가 꺼져 있어도 화면에 알림창을 띄워 알려주는 경보 시스템을 구축합니다.
* **팀 단위 할 일 공유**: 그룹을 생성하여 팀원 간에 할 일을 배정하고 진행률을 공유하는 기능을 제공합니다.
